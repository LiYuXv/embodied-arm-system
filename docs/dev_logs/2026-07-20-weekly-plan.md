# 2026-07-20 至 2026-07-26 本周计划

## 本周目标

本周优先推进以下工作：

1. 修复并完成 Gazebo Classic 中的抓放任务；
2. 完成 LeRobot 基础环境与模型运行链路搭建；
3. 接入一个基础语音识别模块，将识别文本接入现有语言节点；
4. 时间允许时，继续优化或恢复末端夹爪的外观结构。

本计划根据实际调试结果灵活调整，以抓放任务稳定运行和 LeRobot 成功搭建为主要验收目标。

## 每日安排

### 周一

- 回顾 PR #8 和已有抓放代码；
- 复现当前抓放流程中的失败问题；
- 检查工作场景、物体位置、目标区域和抓放配置。

### 周二

- 修复抓取和放置位姿计算；
- 为不同物体分别计算抓取上方、抓取点和抬起点；
- 调整抓放状态机，避免每个动作都重复经过固定的 `pre_pick` 位姿。

### 周三

- 完成红色方块到红色目标区域的完整抓放；
- 验证夹爪开合、抓取、抬升、搬运和释放过程；
- 补充测试和开发记录，准备更新 PR #8。

### 周四

- 建立独立的 LeRobot Python 环境；
- 安装 LeRobot 及必要依赖；
- 运行官方示例或读取公开数据集样本。

### 周五

- 加载一个适合当前硬件条件的 LeRobot 模型；
- 完成一次基础推理；
- 梳理图像、机器人状态、语言指令和动作输出格式，并分析 EDULITE_A3 的适配方式。

### 周六

- 接入基础语音识别模块；
- 将语音识别文本发布为 ROS 2 话题或转交现有语言节点；
- 保留原有终端文字输入方式，并测试常用机械臂指令。

### 周日

- 完成本周功能联调和演示整理；
- 汇总抓放任务、LeRobot 和语音模块的完成情况；
- 时间允许时优化末端夹爪外观结构，并规划下一周任务。

## 本周预期成果

- 红色方块抓放任务能够完整执行；
- PR #8 的遗留抓放问题得到修复或形成明确结论；
- LeRobot 环境能够正常运行，并完成基础模型加载或推理；
- 语音识别结果能够进入现有语言交互流程；
- 更新开发日志和后续任务安排。
